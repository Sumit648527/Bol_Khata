## 📋 Recording Tips

### Visual Flow:
1. **0:00-0:20**: Title slide → Problem statement
2. **0:20-0:35**: Browser → Login → Dashboard
3. **0:35-0:55**: Dashboard features (stay on dashboard)
4. **0:55-1:35**: Voice entry (record, show results, confirm)
5. **1:35-1:50**: Manual entry modal
6. **1:50-2:10**: Customers page
7. **2:10-2:30**: Transactions → Passbook
8. **2:30-2:50**: Architecture (can show diagram or just mention)
9. **2:50-3:00**: Dashboard → Closing

### Recording Setup:
- Use screen recording software (OBS, Loom, or built-in)
- Record at 1080p resolution
- Use a good microphone
- Speak clearly and at moderate pace
- Practice once before final recording
- Keep cursor movements smooth
- Zoom in on important details

### Editing Tips:
- Add subtle background music (low volume)
- Use smooth transitions between sections
- Add text overlays for key points:
  - "Voice-First Technology"
  - "Multi-Language Support"
  - "Real-Time AI Processing"
  - "Production-Ready on AWS"
- Highlight important UI elements with circles/arrows
- Speed up slow parts (like loading) to 1.5x

### What to Emphasize:
1. **Voice feature** - This is your unique selling point
2. **Live production URL** - Shows it's real and deployed
3. **Multi-language support** - Addresses Indian market
4. **Real-time processing** - Shows technical capability
5. **Professional deployment** - HTTPS, domain, AWS

### Common Mistakes to Avoid:
- Don't spend too long on any one feature
- Don't show errors or bugs
- Don't read the script word-for-word (sound natural)
- Don't go over 3 minutes
- Don't forget to show the URL clearly
- Don't use technical jargon without explanation
